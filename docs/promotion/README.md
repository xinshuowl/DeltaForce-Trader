## 推广节奏建议

### 第 1 天 (T0)
- 录好 demo GIF + 4 张截图，提交 (你做)
- 制作 Social Preview 图并上传 GitHub (你做)
- 发 B 站视频 (你做，文案在 `bilibili_video.md`)

### 第 3 天
- B 站视频起量后发 NGA / 小黑盒 / 贴吧

### 第 7 天
- 发技术博文到 V2EX / 掘金，引技术圈流量

### 第 14 天起
- 持续维护 Issue / Discussion
- 跟进每次三角洲行动版本更新发新版

## 风险红线

1. **永远不要**：在文案里用"破解 / 注入 / 外挂 / 黑科技"等词
2. **永远不要**：宣传"绝对不封号" — 这是脚本，永远有风险
3. **永远不要**：在 QQ 群直接发 exe 文件 — 引导到 GitHub Release
4. **建议**：每次推广配合一次免责声明
5. **建议**：核心宣发时间避开三角洲官方大版本更新日 (官方在那时反作弊最严)

## 关键转化漏斗

```
B 站视频 (1k 播放)
    ↓ 5% 点 GitHub 链接
GitHub Repo (50 访客)
    ↓ 50% 点 Star (有 demo + 好的 README)
新增 Stars 25 个
```

> ⚠ 没有 demo gif/视频时这个漏斗的 50% 会掉到 5%，所以**优先做 demo**。
